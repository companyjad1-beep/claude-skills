# UI 공통 규칙 (모든 프로젝트 공통 — 수정하지 말고 그대로 적용)

반복해서 터지는 UI 버그를 사전에 차단하는 고정 규칙. 화면을 만들 때마다 이 파일을 준수한다.

## 레이아웃 겹침 방지

- **하단 네비게이션이 콘텐츠를 가리면 안 된다.** 스크롤 영역의 `padding-bottom = 네비 높이 + env(safe-area-inset-bottom)`. 마지막 아이템이 네비 뒤에 숨는지 항상 확인.
- **상단도 마찬가지.** 고정 헤더가 있으면 콘텐츠에 `padding-top = 헤더 높이 + env(safe-area-inset-top)` (노치·상태바 포함).
- 모바일에서 `100vh` 금지 → `100dvh` 사용 (주소창 때문에 하단이 잘린다).
- 가로 스크롤 유출 금지: flex/grid 자식에 `min-width: 0`, 긴 텍스트는 `truncate`/`ellipsis`.

## z-index 체계 (임의 값 금지)

| 레이어 | 값 |
|--------|-----|
| 고정 헤더/하단 네비 | 10 |
| 드롭다운/팝오버 | 20 |
| 모달 오버레이 | 30 |
| 토스트/알림 | 40 |

`z-index: 9999` 같은 임의 값을 쓰지 않는다. 새 레이어가 필요하면 이 표에 먼저 추가.

## 입력·키보드

- 모바일에서 키보드가 입력창을 가리면 안 된다 (웹: `scrollIntoView` 처리, RN: `KeyboardAvoidingView`).
- iOS 자동 확대 방지: input `font-size`는 16px 이상.
- 터치 타겟 최소 44×44px.

## 상태 처리 (화면마다 필수 4종)

모든 데이터 화면은 **로딩 / 빈 상태 / 에러 / 정상** 4가지 상태를 갖는다. 정상 상태만 만들고 끝내지 않는다.

- 로딩: 스켈레톤 또는 스피너 (레이아웃 점프 방지 — 자리 높이 확보).
- 빈 상태: 안내 문구 + 다음 행동 버튼.
- 에러: 원인 한 줄 + 재시도 버튼.

## 기타 상습 버그

- 버튼 중복 클릭 방지: 제출 중 `disabled` 처리.
- 이미지에 `width/height` 또는 `aspect-ratio` 지정 (레이아웃 점프 방지).
- 다크모드 지원 프로젝트면 색상은 반드시 토큰(변수)으로만 사용 — 하드코딩 hex 금지.
- 날짜·숫자는 로케일 포맷 함수로만 표시.
