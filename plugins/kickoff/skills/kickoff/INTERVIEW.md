# 킥오프 딥 인터뷰 방법론

우로보로스 Socratic Interviewer + grill-me에서 포크한 순수 방법론 (MCP 서버 불필요).

## 인터뷰어 원칙

- **한 번에 질문 하나.** 가장 큰 모호성을 겨냥한 질문 하나만 AskUserQuestion으로 던진다.
- **질문마다 추천안을 함께 제시한다.** 옵션 첫 번째에 "(추천)" 표시. 사용자가 "모르겠다"로 도망칠 수 없게.
- **스스로 답할 수 있는 건 묻지 않는다.**
  - 사실 질문(라이브러리 기능, 가격, API 제약) → 웹 검색으로 직접 확인 후 "~라고 확인했는데 맞나요?"로 통보만.
  - 판단 질문(목표, 범위, 취향, 트레이드오프) → 사용자에게. **의심스러우면 사용자에게 묻는다.**
- **연속 3회 이상 자동 확인으로 넘어가지 않는다.** 3회 연속 사용자 판단 없이 진행했으면 다음 질문은 무조건 사용자에게. (인터뷰 상대는 코드나 문서가 아니라 사람이다.)
- **답변 위에 쌓는다.** 이전 답과 모순되면(예: 출시 2주 + 기능 10개) 그 자리에서 지적하고 조정.
- **초보자 눈높이로 설명한다.** 사용자가 역질문하면("A랑 B 뭐가 나아?", "그게 뭐야?") 옵션부터 다시 들이밀지 말고, 먼저 개념과 차이를 쉬운 말로 설명(필요하면 비교표)한 뒤 추천과 함께 답한다. 전문 용어는 처음 나올 때 한 줄로 풀어준다 (예: "Edge Function = 서버 없이 돌아가는 작은 서버 코드"). 결정에 영향 주는 배경지식은 묻기 전에 먼저 알려준다.
- **존재론적 질문을 활용한다.** "이건 본질적으로 무엇인가?", "그건 원인인가 증상인가?", "지금 우리가 가정하고 있는 건 뭔가?"

## 모호성 장부 (Ambiguity Ledger)

인터뷰 시작 시 아래 5개 트랙을 장부로 만들고, 진행하며 각 트랙의 해소 여부를 추적한다.
한 트랙만 파고들다 나머지를 잊지 말 것 — 몇 라운드마다 장부를 보여주며 남은 트랙을 환기한다.

| 트랙 | 해소 조건 |
|------|----------|
| ① 제품 정의 | 무엇을·누구를 위해·왜, 성공 기준이 한 문장씩 나옴 |
| ② 범위 | MVP 기능 목록 + **명시적 제외 목록**(스코프 크립 방지) + 목표 시점 |
| ③ 기술 스택 | 플랫폼, 프레임워크, DB/인증/결제 등 외부 서비스, 이미 보유한 계정 |
| ④ 브랜드/디자인 | 브랜드 한 줄, 톤앤매너 형용사 3개, 참고 브랜드, 컬러 팔레트, 보이스(말투), 디자인 컨셉, 다크모드, 모바일/데스크톱 우선, 핵심 화면 목록 + 하단 네비 탭 구성 (UI 없으면 트랙 삭제) |
| ⑤ 운영 규칙 | 테스트 정책, 배포처, Claude 금지사항, 커밋 규칙 |

## Refine 게이트 (자유 답변 구조화)

사용자가 이유·제약·범위가 담긴 긴 답을 하면, 한 줄로 압축하지 말고 아래 구조로 정리해 되돌려주고 빠진 것이 없는지 확인받는다:

```
결정: <핵심 결정>
이유: <사용자가 말한 이유>
제약: <명시된 제약>
제외 범위: <명시적으로 뺀 것>
```

"예/아니오"나 고유명사 하나짜리 답은 이 게이트를 건너뛴다. 확인된 내용은 그대로 `docs/DECISIONS.md`의 재료가 된다.

## 종료 조건 (과잉 인터뷰 금지)

- 5개 트랙이 모두 해소되면 **멈춘다.** 문구 다듬기나 좁은 엣지케이스만 남았으면 더 파지 말고 종료를 제안한다.
- 사용자가 "이 정도면 됐다"고 하면 종료 신호로 받아들이고 마지막 확인 질문 하나만 던진다.

## Restate 게이트 (최종 관문)

종료 전, 합의된 내용을 **한 문장 목표**로 압축해 확인받는다:

> "합의한 내용을 한 문장으로 정리하면: `<한 문장>` — 이 문장만 읽은 다른 사람도 같은 결과물을 떠올릴까요?"

옵션: ① 맞다, 문서 생성 시작 ② 문구 수정 ③ 빠진 범위 있음.
②·③이면 수정 내용을 받아 장부에 반영하고 Restate를 다시 한 번만 실행. 두 번 넘게 돌지 않는다 — 그래도 안 맞으면 정체된 트랙에 대한 직접 질문으로 돌아간다.

통과하면 SKILL.md 3단계(문서 생성)로 진행. 한 문장 목표는 PRD 맨 위에 그대로 들어간다.
