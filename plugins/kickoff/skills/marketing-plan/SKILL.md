# Marketing Plan

킥오프가 만든 문서를 재료로 마케팅 기획을 재료 수집 → 인터뷰 → 문서 → 검증 순서로 만든다.
목적: 출시 전에 "누구에게, 뭐라고, 어디서, 언제" 알릴지가 문서로 남는 것. 문서의 모든 실질 문장은 기존 문서·인터뷰 답변·직접 조사한 사실로 추적 가능해야 한다 — 시장 규모·전환율 같은 수치를 지어내면 이 스킬은 실패다.

## 산출물 계약 (완료 시 존재해야 하는 것)

- `docs/MARKETING.md` (템플릿은 아래)
- `docs/decisions/YYYY-MM-DD-마케팅-주제.md` (결정당 파일 하나)
- git 저장소면 커밋 `docs: marketing plan`

## 게이트 (통과하지 못하면 다음 단계 금지)

1. **재료 수집** — `docs/PRD.md` · `docs/BRAND.md` · `docs/decisions/`를 전부 읽는다.
   게이트: 인터뷰 첫 메시지에서 "기존 문서에서 이렇게 확인했다"(타겟·문제·성공 기준·유료화·보이스)를 요약해 보여주고, 이 내용은 다시 묻지 않는다.
   문서가 없으면 kickoff 스킬 먼저 실행을 제안한다. 사용자가 문서 없이 진행을 원하면 제품 한 줄·타겟·유료화 세 가지만 최소로 물어 재료를 채운다.
   **재개**: `docs/decisions/`에 `마케팅-` 주제 파일이 이미 있으면 장부를 복원해 해소된 트랙은 다시 묻지 않고 남은 트랙부터 이어간다.
2. **인터뷰** — [../kickoff/INTERVIEW.md](../kickoff/INTERVIEW.md)의 원칙을 그대로 쓴다: 한 번에 질문 하나 + 추천안, 사실은 조사·판단은 사용자, Refine 게이트 통과 즉시 `docs/decisions/` 기록, 종료 전 Restate 게이트(한 문장 마케팅 목표).
   게이트: 아래 모호성 장부 5트랙 전부 해소 + Restate 통과.

   | 트랙 | 해소 조건 |
   |------|----------|
   | ① 포지셔닝 | 타겟 세그먼트 1개 구체화, 사용자의 현재 대안(경쟁 서비스 또는 수동 해결), 차별점 한 문장 |
   | ② 메시지 | 핵심 소구점 1~2개, 엘리베이터 피치 한 문장, 카피 말투(BRAND 보이스 재사용 — 새로 묻지 않는다) |
   | ③ 채널 | 획득 채널 우선순위 2~3개 + 각 채널을 고른 근거, **명시적 제외 채널**(스코프 크립 방지) |
   | ④ 출시 플랜 | 프리런칭·런칭·포스트런칭 각각 할 일, 시점(PRD 출시 목표와 모순되면 그 자리에서 지적), KPI 1~3개(측정 방법까지) |
   | ⑤ 예산·리소스 | 유료 광고 여부, 쓸 수 있는 돈·주당 시간, 콘텐츠를 누가 만드는가 |

   채널·경쟁 서비스의 사실 관계(가격, 정책, 도달 특성)는 직접 검색해 "~로 확인했는데 맞나요?"로 통보만 한다. 조사한 수치는 출처와 함께만 문서에 넣는다.
3. **문서 생성** — 게이트: `docs/MARKETING.md`의 각 문장이 기존 문서·인터뷰 답변·조사 사실로 추적됨. 안 나온 내용은 비우거나 섹션째 삭제.
4. **최종 검증** — 생성된 파일과 핵심 결정 요약을 장부와 대조해 보여주고 사용자 확인을 받는다. 확인 후 커밋. 파일을 직접 확인하지 않은 채 완료 선언 금지.

## docs/MARKETING.md 템플릿

```markdown
# <프로젝트명> 마케팅 플랜

> 마케팅 목표 한 문장: <Restate 게이트에서 확정된 문장>

## 포지셔닝
- 타겟: <세그먼트 1개, 구체적으로>
- 현재 대안: <경쟁 서비스 또는 지금 해결하는 방식>
- 차별점: <한 문장>

## 메시지
- 엘리베이터 피치: <한 문장>
- 핵심 소구점: <1~2개>
- 카피 말투: <docs/BRAND.md 보이스 준수 — 요약 한 줄>

## 채널 (우선순위 순)
| 채널 | 근거 | 첫 액션 |
|------|------|---------|
| <채널 1> | <왜 이 채널> | <가장 먼저 할 일 하나> |

### 제외 채널 (일부러 안 하는 것)
- <채널> — <뺀 이유>

## 출시 플랜
- 프리런칭 (<시점>): <할 일>
- 런칭 (<시점 — PRD 출시 목표와 정합>): <할 일>
- 포스트런칭 (<시점>): <할 일>

## KPI
- <지표>: <목표치> — 측정: <어디서 어떻게>

## 예산·리소스
- 예산: <금액 / 없음> · 시간: <주당 시간>
- 콘텐츠 제작: <누가>
```

## 경계

- 광고 집행, SNS 계정 생성, 콘텐츠 대량 생산은 범위 밖 — 문서까지만. 완료 후 "첫 액션부터 시작할까요?"로 제안만 한다.
- 제품 정의·범위·브랜드를 바꾸는 답이 나오면 이 스킬에서 수정하지 않는다 — PRD/BRAND의 결정과 모순임을 지적하고, 바꾸려면 해당 문서와 `docs/decisions/`를 먼저 고치게 안내한다.
- 문서는 전부 한국어(서비스명·기술 용어는 영어 그대로).
